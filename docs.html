General

This webservice can be used freely but without guarantee to work. I am keeping it online since more than 2 years now. But downtimes are possible and i am not always home to fix it :) Please do not hesitate to send me ideas to improve the service! segler_alex AT web DOT de PLEASE
Send a descriptive User-Agent in your HTTP requests. This makes it easier for me to get in touch with developers to help with the usage of the API. Something like "appname/appversion" so for example "Cool Radio App/1.2". This also helps me to know which apps are using this service that I can keep the list of apps up to date, so that I can tell people in which ways they can use this service.
If you know an app that is using this service which is not listed on www.radio-browser.info please drop me a note.
Thank you! At the moment this service checks radio stations by connecting to them at least every day and marks them accordingly. (LastCheckOK=0/1)
If you do not find here what you were looking for, please create a feature request on Github or if you are a programmer fork it and make a pull request. Thanks for your help! Recent changes can be found in the Changelog.

List of countries

Will get a JSON encoded list of all countries in the database. If a filter is given, it will only only return the ones that contain the filter as substring, supported output formats: JSON, XML

Syntax:
  • http://www.radio-browser.info/webservice/json/countries
  • http://www.radio-browser.info/webservice/json/countries/<filter>
  • http://www.radio-browser.info/webservice/xml/countries
  • http://www.radio-browser.info/webservice/xml/countries/<filter>
Parameter:
This parameters can be added by using a HTTP POST request with the encoding "x-www-form-urlencoded" instead of GET request. The parameters can also be sent with encoding "application/json" that is used for example by Angular.js
Name Default value Possible values Description
order value value, stationcount name of the attribute the result list will be sorted by
reverse false true, false rerverses the result list if set to true
hidebroken false true, false do not count broken stations
Result JSON:
                [
                {
                  "value": "Albania",
                  "stationcount": "1"
                },
                {
                  "value": "Argentina",
                  "stationcount": "5"
                },
                ..
                ]
Result XML:
                <result>
                <country value="Albania" stationcount="1"/>
                <country value="Argentina" stationcount="5"/>
                ..
                </result>
Example:
  • http://www.radio-browser.info/webservice/json/countries
  • http://www.radio-browser.info/webservice/xml/countries/aus

List of radio stations

Will get a list of radio stations that match the search. The variants with "exact" will only search for perfect matches. The others will match if the station attribute contains the searchterm.
Please use Playable station url API call to get the real station url and also let the click be counted, supported output formats: JSON, XML, M3U, PLS, XSPF, TTL

Syntax:
  • http://www.radio-browser.info/webservice/{format}/stations/byid/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/byuuid/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/byname/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bynameexact/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bycodec/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bycodecexact/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bycountry/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bycountryexact/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bystate/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bystateexact/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bylanguage/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bylanguageexact/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bytag/{searchterm}
  • http://www.radio-browser.info/webservice/{format}/stations/bytagexact/{searchterm}
Parameter:
This parameters can be added by using a HTTP POST request with the encoding "x-www-form-urlencoded" instead of GET request. The parameters can also be sent with encoding "application/json" that is used for example by Angular.js
Name Default value Possible values Description
order name name, url, homepage, favicon, tags, country, state, language, votes, negativevotes, codec, bitrate, lastcheckok, lastchecktime, clicktimestamp, clickcount, clicktrend name of the attribute the result list will be sorted by
reverse false true, false rerverses the result list if set to true
offset 0 0,1,2,3,4,.. starting value of the result list from the database, for example if you want to do paging on the serverside
limit 100000 0,1,2,.... number of returned datarows (stations) starting with offset
Example:
  • http://www.radio-browser.info/webservice/json/stations/byname/jazz
  • http://www.radio-browser.info/webservice/xml/stations/bycountry/austria
  • http://www.radio-browser.info/webservice/xml/stations/bycountryexact/austria
  • http://www.radio-browser.info/webservice/m3u/stations/bytagexact/320kbps
  • http://www.radio-browser.info/webservice/pls/stations/bytagexact/320kbps

Playable station url

Returns the playable url for the station. Any Playlist (PLS, ASX, M3U) decoding will be done for you. This API call will count as a click on the station, supported output formats: JSON, XML ,PLS ,M3U

Syntax:
  • http://www.radio-browser.info/webservice/v2/xml/url/stationid
  • http://www.radio-browser.info/webservice/v2/json/url/stationid
  • http://www.radio-browser.info/webservice/v2/pls/url/stationid
  • http://www.radio-browser.info/webservice/v2/m3u/url/stationid
Result JSON:
          {
            "ok": "true",
            "message": "retrieved station url",
            "id": "123",
            "name": "Station name",
            "url": "http://this.is.an.url"
          }
Result XML:
          <result>
            <status ok="true" message="retrieved station url" id="123" name="Station name" message="http://this.is.an.url"/>
          </result>
Example:
  • http://www.radio-browser.info/webservice/v2/xml/url/123
  • http://www.radio-browser.info/webservice/v2/json/url/123
  • http://www.radio-browser.info/webservice/v2/pls/url/123
  • http://www.radio-browser.info/webservice/v2/m3u/url/123

Advanced station search

Will get a list of radio stations that match the search. It will match if the station attribute contains the searchterm. Please use Playable station url API call to get the real station url and also let the click be counted, supported output formats: JSON, XML, M3U, PLS, XSPF, TTL

Syntax:
  • http://www.radio-browser.info/webservice/xml/stations/search
  • http://www.radio-browser.info/webservice/json/stations/search
Parameter:
This parameters can be added by using a HTTP POST request with the encoding "x-www-form-urlencoded" instead of GET request. The parameters can also be sent with encoding "application/json" that is used for example by Angular.js
Name Default value Possible values Description
name STRING OPTIONAL, name of the station
nameExact false true, false OPTIONAL. True: only exact matches, otherwise all matches.
country STRING OPTIONAL, country of the station
countryExact false true, false OPTIONAL. True: only exact matches, otherwise all matches.
state STRING OPTIONAL, state of the station
stateExact false true, false OPTIONAL. True: only exact matches, otherwise all matches.
language STRING OPTIONAL, language of the station
languageExact false true, false OPTIONAL. True: only exact matches, otherwise all matches.
tag STRING OPTIONAL, a tag of the station
tagExact false true, false OPTIONAL. True: only exact matches, otherwise all matches.
bitrateMin 0 POSITIVE INTEGER OPTIONAL, minimum of kbps for Bitrate field of stations in result
bitrateMax 1000000 POSITIVE INTEGER OPTIONAL, maximum of kbps for Bitrate field of stations in result
order name name, url, homepage, favicon, tags, country, state, language, votes, negativevotes, codec, bitrate, lastcheckok, lastchecktime, clicktimestamp, clickcount, clicktrend OPTIONAL, name of the attribute the result list will be sorted by
reverse false true, false OPTIONAL, rerverses the result list if set to true
offset 0 POSITIVE INTEGER OPTIONAL, starting value of the result list from the database, for example if you want to do paging on the serverside
limit 100000 0,1,2,.... OPTIONAL, number of returned datarows (stations) starting with offset
Example:
  • http://www.radio-browser.info/webservice/json/stations/search
